Recolher as necessidades dos stakeholders para definir as funcionalidades do sistema.

    Process Usage
    • ProjetoSus  >  Warmup  >  Coletar requisitos
    Main Description
    Nesta tarefa, o time identifica os requisitos funcionais e não funcionais do sistema, através de entrevistas, workshops, e análise de documentos. O objetivo é garantir que o software atenda às expectativas dos usuários e outros stakeholders.
    Steps
    Identificar os stakeholders.
    Conduzir entrevistas e reuniões.
    Documentar os requisitos funcionais e não funcionais.
    Validar os requisitos com os stakeholders.
    Priorizar os requisitos para implementação.
