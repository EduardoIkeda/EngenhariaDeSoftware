      Process Usage
      • ProjetoSus  >  Warmup  >  Definir diagrama de caso de uso
      Main Description
      Criar um diagrama que representa as interações dos usuários com o sistema, mapeando as principais funcionalidades e atores envolvidos. Serve como base para o desenvolvimento e testes.
      Steps
      Revisar os requisitos funcionais.
      Identificar atores e suas interações.
      Definir os casos de uso principais.
      Criar o diagrama.
      Revisar com a equipe e stakeholders.
